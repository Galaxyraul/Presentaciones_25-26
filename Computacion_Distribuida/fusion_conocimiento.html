01. Introducción y Contexto

Fusión de Conocimiento en HAR

Integración Invisible y la Oportunidad del Móvil

Sistemas Empotrados y Ubicuos: Integración invisible del procesamiento matemático en el entorno cotidiano de las personas para adquirir y computar magnitudes físicas dinámicas.
El Smartphone como Nodo Sensor: Plataforma inmejorable por su altísima tasa de penetración social, conectividad redundante permanente, poder de cálculo multinúcleo y coprocesadores inerciales de alta precisión.
Actividades Humanas (HAR): Identificación estocástica de estados dinámicos (quieto, caminar, correr, conducir) con alto impacto en e-Health, optimización del rendimiento atlético y movilidad inteligente sostenible.

Estrategia Integradora del Ciclo

Este proyecto no se detiene en la mera captura o el análisis aislado. Diseña e implementa un ciclo de ingeniería completo y desacoplado:

[1] Adquisición: Sensor App móvil nativa.
[2] Persistencia: API PHP y BD MySQL unificada.
[3] Analítica: Dashboard SPA interactivo.

02. Restricciones y Soluciones

Superando el Hosting Compartido

Ingeniería a Medida vs. Dependencias Pesadas

Restricción de Infraestructura: El servidor de hosting compartido carece de permisos de superusuario, impidiendo la instalación del gestor del protocolo AWARE-Light original.
Arquitectura a Medida: Creación de una API REST y esquemas MySQL que emulan con total precisión la interfaz estándar de AWARE, eludiendo la restricción.
Expo & React Native: Adopción de Expo SDK en TypeScript para compilar de forma ágil e inalámbrica a través de servicios cloud (EAS Build), anulando la necesidad de configurar pesados entornos locales de Android Studio.
Decisión Estratégica de Desarrollo

El uso de React Native y TypeScript permite reutilizar el 100% de la lógica tipada tanto en la adquisición nativa móvil como en el parseo del backend, acelerando la depuración y garantizando la coherencia conceptual del protocolo de comunicaciones.

Expo SDK 55 + PHP 8.2 + React 18 SPA

03. Stack de Adquisición

SEU Sensor App: Stack Tecnológico

Robustez Nativa en Primer Plano

Núcleo Nativo: Expo SDK 55 + React Native 0.83.6 implementando tipos estrictos en TypeScript 5.9.2 para evitar comportamientos nulos o indefinidos en la captura.

Garantía de Persistencia Local

Para garantizar que el sistema operativo no congele ni mate la aplicación móvil durante las caminatas prolongadas, se incorporaron dos mecanismos críticos:

[A] AsyncStorage: Resguarda de forma local el UUID único de dispositivo y estados de etiqueta en caso de reinicio forzado.
[B] Foreground Service: La librería `react-native-background-actions` inyecta un hilo permanente con notificación en la barra de tareas, evitando que Android reclame los recursos del sistema.

03. Optimización del Pipeline

Desacoplamiento de Bucles

Evitando la Saturación del Hilo de Renderizado (UI Thread)

El Desafío: Muestrear sensores a 50 Hz provoca 50 disparos de eventos por segundo. Actualizar el estado de React con cada disparo satura por completo el hilo principal de renderizado (UI Thread), congelando la aplicación y tirando la tasa de fotogramas a menos de 5 FPS.

Arquitectura de Doble Bucle Desacoplado: Bucle 1 (Adquisición - 50 Hz): Escribe de forma directa en variables mutables en memoria RAM (`useRef`), consumiendo menos de un 5% de procesador.
Bucle 2 (Render UI - 10 Hz): Un temporizador desacoplado lee las variables de memoria cada 100 ms solo para pintar las métricas en la interfaz, **reduciendo la carga en UI en un 80%**.

03. Marco Físico y Científico

Sensores Utilizados y Muestreo

Fundamentos del Reconocimiento de Actividades

Movimiento e Impacto

Acelerómetro Triaxial

Mide la aceleración física aplicada sobre el dispositivo en los tres ejes espaciales. Permite identificar la periodicidad del paso del usuario y la amplitud de los impactos para discriminar con precisión entre caminar y correr.

\vec{a} = (a_x, a_y, a_z) \quad \text{Magnitude: } ||a|| = \sqrt{a_x^2 + a_y^2 + a_z^2}
Rotación y Orientación

Giroscopio de Precisión

Registra la velocidad angular del chasis móvil. Esencial para caracterizar giros complejos, pérdidas de equilibrio o dinámicas de rotación que el acelerómetro no puede aislar (como subir/bajar escaleras).

\vec{\omega} = (\omega_x, \omega_y, \omega_z) \quad (rad/s)
Cambios de Altitud

Sensor de Presión Barométrica

Monitorea los cambios de presión atmosférica ambiental a nivel nativo. Posibilita calcular altimetrías relativas con margen de centímetros, detectando ascensos verticales mecánicos (como en un ascensor) imperceptibles para el GPS.

\Delta h \approx 44330 \times \left(1 - \left(\frac{P}{P_0}\right)^{\frac{1}{5.255}}\right)
Orientación Absoluta

Magnetómetro de Silicio

Registra la intensidad del campo magnético circundante en microteslas. Proporciona la orientación absoluta respecto al norte magnético y permite geolocalización interior mediante firmas ferromagnéticas en edificios.

\vec{B} = (B_x, B_y, B_z) \quad (\mu T)
Posicionamiento y Velocidad

Geolocalización por Satélite

Entrega coordenadas espaciales y velocidad de desplazamiento a 1 Hz. Es la variable fundamental de desambiguación para separar desplazamientos vehiculares rápidos de carreras pedestres de alta intensidad.

v_{lineal} = \frac{d(P_1, P_2)}{\Delta t} \quad \text{(Haversine Distance)}

04. Diseño de Base de Datos

Modelo de Datos y Sincronización

Garantizando Rendimiento y Autonomía de Batería

Estructura Normalizada: Siguiendo el protocolo AWARE, se divide el modelo en una tabla de metadatos de dispositivos y tablas de lectura segregadas por sensor para eludir colisiones de tipos de datos.
Índices Compuestos: Inclusión crítica de un índice compuesto `(device_id, timestamp)` en todas las tablas de lectura analíticas, impidiendo lecturas secuenciales de tabla completa (*table scans*) durante las consultas de series temporales del Dashboard.
Sincronización por Lotes: Para salvaguardar la autonomía de la batería y la red celular del móvil, los registros se acumulan localmente en la RAM y se envían asíncronamente en paquetes JSON de gran tamaño cada 30 segundos.

Esquema de Índices Optimizados

Las tablas analíticas como `sensor_accelerometer` crecen de forma exponencial. Sin índices adecuados, las lecturas colapsan el backend: 

CREATE TABLE sensor_accelerometer (
    id INT AUTO_INCREMENT PRIMARY KEY,
    device_id VARCHAR(50),
    timestamp DOUBLE,
    double_x DOUBLE,
    double_y DOUBLE,
    double_z DOUBLE,
    label VARCHAR(30),
    INDEX idx_dev_time (device_id, timestamp)
);

04. Backend y API

Backend Extensible en PHP

Inyección Segura y Enrutamiento Eficiente

Punto de Entrada Único: Archivo `index.php` auto-enrutable mediante parámetros que discrimina de forma automática entre la ingesta masiva JSON del móvil y las consultas de consulta analítica solicitadas por el cliente React.
Sentencias Preparadas PDO: Vinculación estricta de parámetros en MySQL para neutralizar vulnerabilidades críticas por inyección de código SQL malicioso.
Lista Blanca de Sensores: Un middleware de validación selectiva intercepta las cargas y descarta flujos no registrados, blindando la integridad física de la base de datos.

Protección contra Falsificación

Dado que los smartphones envían información de forma directa por red celular expuesta, se inyecta un encabezado de autorización basado en firma compartida (*Shared Secret*) de estudio para el registro inicial de nuevos nodos sensores en la tabla maestra.

if ($_SERVER['HTTP_X_STUDY_AUTH'] !== STUDY_SECRET) {
    http_response_code(401);
    exit("No autorizado");
}

06. Experimentación de Campo

Protocolo de Recogida Experimental

Metodología y Control Dinámico

Hardware Homogéneo de Prueba: Despliegue simultáneo del prototipo de adquisición móvil en tres terminales idénticos (Samsung Galaxy S20 FE, Android 13, procesadores Snapdragon 865).
Metodología Rigurosa de Campo: Sesiones continuadas de adquisición con una duración fija de al menos 5 minutos para cada una de las actividades definidas en el estudio.
Etiquetado de Ground Truth: El propio usuario móvil ingresa de forma activa su estado actual en la pantalla de la SEU Sensor App previo a iniciar la marcha.

Simplificación de Pipeline Analítico

Al incrustar la etiqueta de Ground Truth (*label*) directamente en caliente en la estructura interna de cada muestra en el hilo nativo de captura del móvil, se descarta por completo la necesidad de realizar alineaciones temporales manuales complejas durante la fase de análisis del dataset.

07. Explotación Analítica

Consultas de Fusión de Conocimiento

Demostración Práctica del Potencial SQL en HAR

Caracterización de Firma Física

Fusión del Acelerómetro Triaxial

Calcula la aceleración resultante eliminando el factor del sesgo por orientación espacial del dispositivo, facilitando la identificación inmediata de la intensidad dinámica. 

SELECT label, 
       AVG(SQRT(double_x*double_x + double_y*double_y + double_z*double_z)) AS magnitude_avg, 
       VARIANCE(SQRT(double_x*double_x + double_y*double_y + double_z*double_z)) AS magnitude_var
FROM sensor_accelerometer
GROUP BY label;
Análisis Espacio-Temporal

Reconstrucción Cronológica GPS

Combina velocidad, geoposicionamiento y marcas temporales para modelar trayectorias físicas completas y aislar aceleraciones externas. 

SELECT timestamp, latitude, longitude, speed, label
FROM sensor_gps
WHERE device_id = 'S20FE_01'
ORDER BY timestamp ASC;
Detección de Movimientos Verticales

Variación Atmosférica

Agrupa lecturas del barómetro en tramos temporales consecutivos para estimar velocidades de elevación vertical y correlacionar subida de rampas. 

SELECT ROUND(timestamp/10) * 10 AS time_block,
       (MAX(pressure) - MIN(pressure)) AS pressure_delta,
       AVG(pressure) AS pressure_avg
FROM sensor_barometer
GROUP BY time_block
ORDER BY time_block ASC;
