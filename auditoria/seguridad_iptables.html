Objetivo de la Auditoría

Protección del Host

Evaluar y aplicar metodologías de desarrollo seguro para mitigar riesgos en la infraestructura perimetral del sistema operativo.

  • Reducción de la superficie de ataque.
  • Implementación del principio de Mínimo Privilegio.
  • Garantizar el cumplimiento de normativas de seguridad (ISO 27001).
  • Establecer trazas de auditoría mediante logs.
03 / 11

Fundamentos: Netfilter

Framework del Kernel

Netfilter permite:

  • Filtrado de paquetes.
  • Traducción de direcciones de red (NAT).
  • Manipulación de paquetes (Mangle).

IPTables actúa como la herramienta de administración en el espacio de usuario.

Operación a nivel de Kernel (Capa 3 y 4 del modelo OSI)

05 / 11

Metodología: Default Deny

Premisa de seguridad absoluta

Política por Defecto (DROP): Se descarta todo paquete que no coincida explícitamente.

iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT DROP

Garantiza que un servicio no sea visible por defecto.

06 / 11

Stateful Inspection

Módulo conntrack

  • NEW: Conexión iniciada por el cliente.
  • ESTABLISHED: Sesión activa validada.
  • RELATED: Conexiones secundarias (FTP, ICMP).
iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
07 / 11

Caso Práctico: Web & SSH

Configuración en producción

# Permitir loopback (interno)
iptables -A INPUT -i lo -j ACCEPT

# Abrir tráfico Web público
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Administración SSH restringida por IP
iptables -A INPUT -p tcp -s 192.168.1.100 --dport 22 -j ACCEPT

Trazabilidad y Logging

Registro de accesos

iptables -A INPUT -m limit --limit 5/min -j LOG --log-prefix "FW_REJECT: "
  • Identificación de ataques de fuerza bruta.
  • Análisis forense tras incidentes.
  • Monitorización de políticas de acceso.
