¿Qué es FP-Growth?

Frequent Pattern Growth

Definición

Es un algoritmo de minería de reglas de asociación que encuentra conjuntos de elementos frecuentes sin necesidad de generar candidatos explícitos.

Compresión

Usa una estructura de árbol para resumir la base de datos.

Divide & Conquer

Fragmenta la búsqueda en sub-problemas menores.

Fase 1: Construcción

Dos Pasos sobre la Base de Datos

1
Conteo y Filtrado: Se escanea la BD para contar la frecuencia de cada item. Se eliminan los que no cumplen el support mínimo.
2
Inserción Ordenada: Se escanea la BD de nuevo. Para cada transacción, se ordenan sus items (frecuencia descendente) y se insertan en el árbol.
06 / 10

Fase 2: Minería

Extrayendo el Conocimiento

Algoritmo Recursivo

El algoritmo busca patrones desde el item menos frecuente hacia la raíz (enfoque bottom-up).

  • Se genera la Base de Patrones Condicional para cada item.
  • Se construye un Árbol FP Condicional específico.
  • Se repite el proceso hasta que no se puedan encontrar más patrones frecuentes.

Conclusiones

Cierre de la Sesión

  • FP-Growth es el estándar de oro para bases de datos masivas.
  • La eliminación de candidatos reduce drásticamente el uso de memoria.
